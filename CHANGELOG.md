# 更新日志

本项目遵循语义化版本号。

## [0.7.0] - 2026-07-13

### 新增

- 添加 SSH 策略、监听端口、待更新软件包和 Docker 公开端口安全体检。
- 添加 UFW 状态、安装和安全启用。
- 添加 Fail2ban 状态及 sshd jail 安装配置。
- 添加 Fail2ban 配置备份、语法校验和失败回滚。
- 添加安全中心静态测试。

### 安全

- UFW 启用前先允许自动检测到的 SSH TCP 端口，并要求输入 `ENABLE`。
- 不提供 UFW 禁用、重置或任意开放端口功能。
- 不修改 root SSH 登录、密码认证或其他 OpenSSH 配置。

## [0.6.0] - 2026-07-13

### 新增

- 添加 `winalong_apps` 用户自定义 bridge 网络和现有应用接入功能。
- 新部署应用自动加入专用网络，可通过容器名进行内部监控。
- 添加 Nginx 与 Uptime Kuma 命名数据卷一致性备份。
- 添加备份列表、归档校验和最新备份恢复。
- 添加恢复前安全快照及失败自动回滚。
- 添加备份恢复静态安全测试。

### 安全

- 恢复必须输入完整 `RESTORE` 确认短语。
- 数据清理仅在临时 Alpine 容器的 `/target` 数据卷挂载点执行。
- 不提供 Docker 数据卷删除或备份文件自动清理功能。

## [0.5.0] - 2026-07-13

### 新增

- 添加 Docker 应用中心及独立应用目录配置。
- 添加 Nginx 官方稳定 Alpine 镜像部署与管理。
- 添加 Uptime Kuma v2 官方镜像部署与管理。
- 添加应用状态、启停、重启、日志及 SSH 隧道访问说明。
- 添加应用中心静态安全测试。
- 完善 Portainer 首次初始化时限与 Setup token 安全提示。

### 安全

- 所有应用默认仅绑定 `127.0.0.1`。
- 禁止 host 网络、公开绑定和应用数据删除操作。
- AdGuard Home 因涉及 DNS 53 端口与网络接管，未纳入本版本。

## [0.4.0] - 2026-07-13

### 新增

- 添加 Portainer CE LTS 部署、状态、日志及启停管理。
- 添加 Docker Compose 项目查询。
- 添加 Portainer SSH 隧道访问说明。
- 添加 Portainer 静态安全测试。

### 安全

- Portainer 默认仅绑定 `127.0.0.1:9443`，不公开管理界面。
- 不启用仅供 Edge Agent 使用的 8000 端口。
- 不提供 Portainer 容器或数据卷删除功能。

## [0.3.0] - 2026-07-13

### 新增

- 添加 Docker 状态、版本、容器、镜像与磁盘占用查询。
- 添加基于 Docker 官方 APT 仓库的 Engine 与 Compose 插件安装。
- 添加经 root 权限检查和二次确认的 Docker 服务启停与重启。
- 添加 Docker 模块静态安全测试，阻止引入数据删除操作。

### 安全

- 不开放端口、不创建应用容器、不清理 Docker 数据，也不自动修改用户组。

## [0.2.0] - 2026-07-13

### 新增

- 添加 Ubuntu/Debian 软件包更新检查与常规更新安装。
- 添加 systemd 时间同步状态检查与启用操作。
- 添加可配置大小的 Swap 创建功能，包含重复检测、二次确认和失败清理。
- 所有新增系统写操作均要求 root 权限并记录日志。

## [0.1.0] - 2026-07-12

### 新增

- 初始化可维护的 Bash 项目骨架。
- 添加循环主菜单和系统检查子菜单。
- 添加系统信息、BBR 状态和 Swap 状态只读检查。
- 添加公共函数、统一 UI、默认配置和日志能力。
- 添加幂等友好的安装脚本与保留日志、备份的卸载脚本。
- 添加关键文件、Bash 语法和可选 ShellCheck 烟雾测试。
- 添加 README、架构说明、忽略规则及运行数据占位目录。
- 固定 Shell 等文本文件使用 LF 换行，便于在 Windows 开发、Linux 运行。

### 修复

- 修复通过 `/usr/local/bin/winalong` 软链接启动时无法定位配置文件的问题。
