# 更新日志

本项目遵循语义化版本号。

## [1.1.2] - 2026-07-13

### 修复

- 烟雾测试先完整捕获 CLI 帮助文本再匹配入口，避免 `pipefail` 与 `grep -q` 引发偶发 SIGPIPE 误报。

## [1.1.1] - 2026-07-13

### 修复

- ShellCheck 现在固定从项目根目录执行，修复在线更新从其他工作目录启动时产生的 `SC1091` 误报。
- 保持远程测试失败时不修改当前安装的安全行为。

## [1.1.0] - 2026-07-13

### 新增

- 添加更新后维护状态，显示可升级和暂缓软件包。
- 添加 `/var/run/reboot-required`、运行内核、最新已安装内核和失败 systemd 单元检查。
- 健康体检新增独立重启状态，并区分少量更新和暂缓更新。
- 添加 `winalong --maintenance` 非交互维护检查入口。
- 添加维护状态静态安全测试。

### 安全

- 暂缓软件包仅通过 `apt-get -s upgrade` 模拟检测。
- 不自动执行 `full-upgrade`、`dist-upgrade`、发行版升级或服务器重启。

## [1.0.0] - 2026-07-13

### 正式版

- 添加支持系统、磁盘、内存、负载、时间同步、软件更新、安全防护和 BBR 的百分制健康体检。
- 添加健康建议与 Docker 运行状态摘要；未安装可选 Docker 功能不扣分。
- 添加 `--doctor`、`--version` 和 `--help` 非交互命令行入口。
- 将在线更新通道从 beta 切换为 stable。
- 添加健康体检静态安全测试和正式版发布验收文档。

### 安全

- 健康体检保持完全只读，不执行自动修复、服务变更或软件安装。
- 延续不修改 SSH、不开放任意端口、不执行远程脚本管道的正式版安全边界。

## [0.9.2] - 2026-07-13

### 修复

- 修复插件安装所有权回归测试的 ShellCheck `SC2016` 提示，确保在线更新的远程烟雾测试可以通过。

## [0.9.1] - 2026-07-13

### 修复

- 安装时将 `/opt/winalong-toolbox` 中的程序和内置插件明确设为 `root:root`，避免从普通用户仓库复制后被插件安全校验拒绝。
- 管理员插件目录在安装时明确使用 root 所有权。

## [0.9.0] - 2026-07-13

### 新增

- 添加基于 GitHub SSH 的在线版本检查和更新入口。
- 添加更新前远程烟雾测试、安装目录快照及失败自动回滚。
- 添加拒绝降级、固定安装目录和完整 `UPDATE` 确认保护。
- 添加插件列表、权限验证、SHA-256 展示和完整 `RUN` 确认。
- 添加只读系统摘要示例插件及持久化管理员插件目录。
- 添加在线更新与插件框架静态安全测试。

### 安全

- 在线更新不执行 `curl | bash` 或 `wget | bash`，仅从配置的 Git SSH 仓库暂存代码。
- 插件必须由 root 所有，且组和其他用户不可写；插件不会在启动时自动加载。
- 卸载时保留日志、备份和管理员自定义插件。

## [0.8.0] - 2026-07-13

### 新增

- 添加网络接口、IPv4/IPv6 路由与 DNS 概况。
- 添加固定目标 Ping、DNS、MTR 和 Traceroute 诊断。
- 添加 Cloudflare 25 MB 限量下载测速及隐私提示。
- 添加 BBR、可用拥塞控制、qdisc 和模块状态检查。
- 添加 BBR/fq 持久配置、旧文件备份及失败回滚。
- 添加网络诊断静态安全测试。

### 安全

- 不执行任何远程一键脚本，不修改系统路由。
- BBR 启用要求输入完整 `ENABLE`，确认前不加载模块或修改 sysctl。
- 测速目标和诊断地址来自固定配置，不接受交互式命令参数。

## [0.7.0] - 2026-07-13

### 新增

- 添加 SSH 策略、监听端口、待更新软件包和 Docker 公开端口安全体检。
- 添加 UFW 状态、安装和安全启用。
- 添加 Fail2ban 状态及 sshd jail 安装配置。
- 添加 Fail2ban 配置备份、语法校验和失败回滚。
- 添加安全中心静态测试。

### 安全

- UFW 启用前先允许自动检测到的 SSH TCP 端口，并要求输入 `ENABLE`。
- 不提供 UFW 禁用、重置或任意开放端口功能。
- 不修改 root SSH 登录、密码认证或其他 OpenSSH 配置。

## [0.6.0] - 2026-07-13

### 新增

- 添加 `winalong_apps` 用户自定义 bridge 网络和现有应用接入功能。
- 新部署应用自动加入专用网络，可通过容器名进行内部监控。
- 添加 Nginx 与 Uptime Kuma 命名数据卷一致性备份。
- 添加备份列表、归档校验和最新备份恢复。
- 添加恢复前安全快照及失败自动回滚。
- 添加备份恢复静态安全测试。

### 安全

- 恢复必须输入完整 `RESTORE` 确认短语。
- 数据清理仅在临时 Alpine 容器的 `/target` 数据卷挂载点执行。
- 不提供 Docker 数据卷删除或备份文件自动清理功能。

## [0.5.0] - 2026-07-13

### 新增

- 添加 Docker 应用中心及独立应用目录配置。
- 添加 Nginx 官方稳定 Alpine 镜像部署与管理。
- 添加 Uptime Kuma v2 官方镜像部署与管理。
- 添加应用状态、启停、重启、日志及 SSH 隧道访问说明。
- 添加应用中心静态安全测试。
- 完善 Portainer 首次初始化时限与 Setup token 安全提示。

### 安全

- 所有应用默认仅绑定 `127.0.0.1`。
- 禁止 host 网络、公开绑定和应用数据删除操作。
- AdGuard Home 因涉及 DNS 53 端口与网络接管，未纳入本版本。

## [0.4.0] - 2026-07-13

### 新增

- 添加 Portainer CE LTS 部署、状态、日志及启停管理。
- 添加 Docker Compose 项目查询。
- 添加 Portainer SSH 隧道访问说明。
- 添加 Portainer 静态安全测试。

### 安全

- Portainer 默认仅绑定 `127.0.0.1:9443`，不公开管理界面。
- 不启用仅供 Edge Agent 使用的 8000 端口。
- 不提供 Portainer 容器或数据卷删除功能。

## [0.3.0] - 2026-07-13

### 新增

- 添加 Docker 状态、版本、容器、镜像与磁盘占用查询。
- 添加基于 Docker 官方 APT 仓库的 Engine 与 Compose 插件安装。
- 添加经 root 权限检查和二次确认的 Docker 服务启停与重启。
- 添加 Docker 模块静态安全测试，阻止引入数据删除操作。

### 安全

- 不开放端口、不创建应用容器、不清理 Docker 数据，也不自动修改用户组。

## [0.2.0] - 2026-07-13

### 新增

- 添加 Ubuntu/Debian 软件包更新检查与常规更新安装。
- 添加 systemd 时间同步状态检查与启用操作。
- 添加可配置大小的 Swap 创建功能，包含重复检测、二次确认和失败清理。
- 所有新增系统写操作均要求 root 权限并记录日志。

## [0.1.0] - 2026-07-12

### 新增

- 初始化可维护的 Bash 项目骨架。
- 添加循环主菜单和系统检查子菜单。
- 添加系统信息、BBR 状态和 Swap 状态只读检查。
- 添加公共函数、统一 UI、默认配置和日志能力。
- 添加幂等友好的安装脚本与保留日志、备份的卸载脚本。
- 添加关键文件、Bash 语法和可选 ShellCheck 烟雾测试。
- 添加 README、架构说明、忽略规则及运行数据占位目录。
- 固定 Shell 等文本文件使用 LF 换行，便于在 Windows 开发、Linux 运行。

### 修复

- 修复通过 `/usr/local/bin/winalong` 软链接启动时无法定位配置文件的问题。
