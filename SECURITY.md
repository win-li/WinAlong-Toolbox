# 安全策略

## 支持版本

当前仅为最新稳定版本提供安全修复。请先通过 `winalong` 的在线更新功能升级，再报告仍可复现的问题。

## 报告安全问题

请不要在公开 Issue 中发布密码、Token、私钥、服务器 IP、日志中的凭据或可直接利用的漏洞细节。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口（仓库所有者需先启用 Private vulnerability reporting）。如果该入口不可用，请通过仓库所有者提供的私密联系方式报告；在确认安全问题已修复前，请勿公开利用细节。

报告中建议包含受影响版本、操作系统、最小复现步骤、实际影响及已做的脱敏处理。项目不会要求你提供 SSH 密码或私钥。

## 安全边界

WinAlong Toolbox 会执行部分需要 root 权限的系统管理操作。运行前请保留云平台快照和现有 SSH 会话。`curl | bash` 安装会信任当时从 GitHub 获取的引导脚本；安全要求较高时，请使用克隆仓库、审阅代码、运行测试后再安装的方式。
