# 备份生命周期与配置安全

v1.4.0 为 Nginx 与 Uptime Kuma 备份增加健康检查、完整性校验、保留策略预览和显式清理。

- 应用归档只接受项目固定命名格式，且必须是备份目录中的普通文件。
- 默认每个应用保留最新 7 个归档，最低不得少于 3 个。
- 清理永不自动运行；先显示预览，再要求输入完整的 `PURGE`。
- 符号链接、非标准文件名和备份目录外的路径不会被删除。
- 创建新备份前要求目标磁盘至少保留 10% 可用空间。
- 完整性校验会读取所有标准 `tar.gz`，成功后生成权限为 `600` 的 SHA-256 清单。

托管配置快照只收集 WinAlong 默认配置和本项目创建的少量 systemd、BBR、Fail2ban 文件。它不包含 `config/local.conf`、环境变量、密钥、SSH 配置、UFW 规则详情、管理员插件或日志，也不提供自动恢复。

脱敏支持包包含诊断报告、备份健康摘要、清单和校验和。支持包不包含原始日志或 Docker inspect 输出；分享前仍应人工检查。
